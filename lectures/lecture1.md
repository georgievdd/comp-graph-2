## Гистограмма яркости

**Определение:** столбчатая диаграмма — для каждой градации яркости показывает, сколько пикселей с этой яркостью есть в изображении.

```
H(j) = количество пикселей с яркостью j,  j ∈ [0, Bmax]
```

**Пример:** изображение 5×5, значения 0..4:
```
1 0 4 0 2 3
3 2 1 0 1 3
0 2 0 1 1 0
4 2 2 1 2 0
1 2 2 1 1 3
```
→ H(0)=7, H(1)=9, H(2)=8, H(3)=4, H(4)=2

**Плотность вероятности:**
```
P(j) = H(j) / N,  где N — общее число пикселей
```
P(j) — вероятность того, что случайный пиксель имеет яркость j.

**Полулогарифмический масштаб:** если какое-то значение доминирует (огромный пик), строят `log10(H(j))` по оси Y, чтобы увидеть остальное распределение.

**Для цветных изображений:** строят отдельные гистограммы для каналов R, G, B. Также возможны **многомерные гистограммы** (например, 2D по каналам RB) — нормированные трактуются как условные вероятности.

### Код: гистограмма полутонового изображения 8 bpp (C++)

```cpp
void Histogram4Gray8bpp(const unsigned char* pbImg, int iHeight, int iWidth,
                         int iWidthBytes, unsigned long* pulHist)
{
    memset(pulHist, 0, 256 * sizeof(*pulHist));
    for (int y = 0; y < iHeight; ++y)
        for (int x = 0; x < iWidth; ++x)
            pulHist[ pbImg[iWidthBytes * y + x] ]++;
}
```

`iWidthBytes` — шаг строки в байтах (может быть > iWidth из-за выравнивания).

## Гистограммные признаки (статистические моменты)

Гистограмма трактуется как распределение вероятностей → используются стандартные статистические признаки.

### Среднее (математическое ожидание / mean)

```
Ī = Σ(j=0..Bmax) j · P(j)
```

Характеризует среднюю яркость изображения.

### Дисперсия (variance) и СКО

```
σ² = Σ(j=0..Bmax) (j − Ī)² · P(j)
σ = √σ²  — среднеквадратичное отклонение
```

Мера разброса значений яркости (контрастность).

### Коэффициент эксцесса (kurtosis)

```
E = (1/σ⁴) · Σ(j=0..Bmax) (j − Ī)⁴ · P(j)  − 3
```

Мера остроты пика распределения:
- `E > 0` — острее гауссовой (лептокуртозис)
- `E = 0` — гауссово распределение
- `E < 0` — более плоское (платикуртозис)

### Коэффициент асимметрии (skewness)

```
A = (1/σ³) · Σ(j=0..Bmax) (j − Ī)³ · P(j)
```

Характеризует «скошенность» (несимметричность):
- `A > 0` — хвост вправо
- `A < 0` — хвост влево

### Энергия / однородность (Uniformity)

```
En = Σ(j=0..Bmax) P²(j)
```

- Минимальна для **равномерной** гистограммы
- Максимальна, когда все пиксели одной яркости
- Характеризует наличие пиков

### Энтропия (Entropy)

```
Hn = − Σ(j=0..Bmax) P(j) · log₂(P(j))
```

- Максимальна для **равномерной** гистограммы
- Минимальна при наличии пиков
- Пример: однородное изображение → Hn≈8.0, Eₙ≈0.004; текстурное → Hn≈5.4, Eₙ≈0.087

### Квартили и перцентили

Гистограмма как CDF (накопленная вероятность):

| Перцентиль | Квартиль | Смысл |
|---|---|---|
| 0-й | — | минимум |
| 25-й | 1-й (Q1) | — |
| 50-й | 2-й (Q2) | медиана |
| 75-й | 3-й (Q3) | — |
| 100-й | 4-й | максимум |

**Межквартильный размах** = Q3 − Q1 (устойчивая к выбросам мера разброса)

**Пример разбора:** по (mean, σ) определить, какому изображению соответствуют значения:
- mean=145.9, σ=85.6 → светлое с сильным разбросом
- mean=146.9, σ=51.4 → светлое с умеренным разбросом
- mean=62.8, σ=67.8 → тёмное с сильным разбросом

## Матрица совместной встречаемости (GLCM)

**Gray Level Co-occurrence Matrix** (также: матрица вхождений, гистограмма второго порядка).

### Определение

Содержит количество совместных появлений пар значений пикселей на заданном расстоянии `(dr, dc)`.

```
P(dr,dc)(a, b) = Ns(a,b) / Nt
```

- `Ns(a,b)` — число пар пикселей: I(r,c)=a И I(r+dr, c+dc)=b
- `Nt` — общее число таких пар
- `a, b ∈ [0, Bmax]`

**Смысл:** условная вероятность того, что пиксель имеет значение `a`, при условии что на dr строк и dc столбцов от него находится пиксель со значением `b`.

### Пример для dr=1, dc=0

```
Изображение:        GLCM (строки=b, столбцы=a):
1 2 0 1 0             a:  0 1 2 3
3 2 1 0 0          b: 0 [ 3 2 0 0 ]
1 1 2 3 0             1 [ 1 0 1 4 ]
3 3 3 1 0             2 [ 0 1 1 1 ]
1 2 1 0 3             3 [ 2 3 1 0 ]
```

Разные `(dr, dc)` → разные матрицы.

### Визуальная интерпретация

| Тип изображения | Вид GLCM |
|---|---|
| Фотореалистичное | Значения вытянуты вдоль главной диагонали (соседние пиксели близки по яркости) |
| Идеальная текстура | Один или несколько резких пиков (регулярный рисунок) |
| Естественная текстура | Несколько сгруппированных пиков |

### Текстурные признаки по GLCM

```
Автокорреляция:   BA = ΣΣ a·b·P(a,b)
Ковариация:       BC = ΣΣ (a−ā)·(b−b̄)·P(a,b)
Контраст (момент инерции): BI = ΣΣ (a−b)²·P(a,b)
Энергия:          BE = ΣΣ P²(a,b)
Энтропия:         BN = −ΣΣ P(a,b)·log₂(P(a,b))
```

Используются для детектирования и классификации текстур.

### Расширения GLCM

- 2D и многомерные матрицы для цветных изображений
- **SCM (Structural Co-occurrence Matrix)** — строится для пары (значение пикселя, значение после фильтрации); семейство матриц по набору фильтров

## Проекции

**Определение:** сумма пикселей на линии, перпендикулярной данной оси.

```
Проекция на вертикальную ось:    Pv(r) = Σ(c) I(r,c)
Проекция на горизонтальную ось:  Ph(c) = Σ(r) I(r,c)
```

**Пример:**
```
1 0 1 0  →  Pv: 2
0 0 0 0  →      0
1 1 1 1  →      4
0 0 1 0  →      1
1 1 1 0  →      3
Ph:  3 2 4 1
```

**Применения:**
- Анализ сканированных документов (нахождение строк, колонок)
- Восстановление изображения (томография): в тривиальных случаях — несколько проекций; для полутоновых — тысячи (метод обратного проецирования)

## Профили интенсивности

**Определение:** значения пикселей вдоль прямой (иногда кривой) линии на изображении.

Строят профили яркости и профили цветовых каналов.

**Применения:**
- Оценка резкости изображения (скорость перепада на крае)
- Оценка равномерности освещённости фона

## Показатели качества изображений

### Классификация

```
Показатели качества
├── Качественные (субъективные) — экспертная оценка человеком
└── Количественные (объективные)
    ├── Безреференсные (no-reference / blind) — без эталона
    └── Референсные — сравнение с эталонным изображением
```

Цель — чтобы количественные показатели согласовывались с субъективными оценками.

---

### Референсные показатели

#### MSE (Mean Square Error) — среднеквадратичная ошибка

Для изображений I и Iₙ размера M×N:

```
MSE = (1/MN) · ΣΣ |I(r,c) − Iₙ(r,c)|²
```

#### SNR (Signal-to-Noise Ratio) — отношение сигнал/шум

```
SNR = μ_sig / MSE

SNR = 10·log₁₀( ΣΣ I(r,c)² / ΣΣ |I(r,c)−Iₙ(r,c)|² )  [дБ]
```

#### PSNR (Peak Signal-to-Noise Ratio) — пиковое ОСШ

```
PSNR = 10·log₁₀(MAX_I² / MSE) = 20·log₁₀(MAX_I / √MSE)  [дБ]
```

`MAX_I` — максимально возможное значение пикселя (для 8 bpp = 255).

**Типичные значения PSNR:**
- > 40 дБ — высокое качество
- 30–40 дБ — приемлемое
- < 30 дБ — заметные искажения

#### Недостатки PSNR

**Нельзя** сравнивать разные изображения или разные типы искажений по MSE/PSNR. Три изображения с одинаковым PSNR=28 дБ могут выглядеть принципиально по-разному:
- гауссовый шум (AWGN σ=10)
- артефакты сжатия
- пониженный контраст

### SSIM (Structural Similarity Index)

Один из лучших референсных показателей качества по согласованности с оценками человека.

Вычисляется как взвешенное произведение трёх компонент:

```
SSIM(x, y) = l(x,y)^α · c(x,y)^β · s(x,y)^γ
```

| Компонента | Формула | Смысл |
|---|---|---|
| Яркость `l` | (2μₓμᵧ + c₁) / (μₓ² + μᵧ² + c₁) | сравнение средних |
| Контраст `c` | (2σₓσᵧ + c₂) / (σₓ² + σᵧ² + c₂) | сравнение СКО |
| Структура `s` | (σₓᵧ + c₃) / (σₓσᵧ + c₃) | нормированная ковариация |

Константы стабилизации:
```
c₁ = (0.01·L)²,  c₂ = (0.03·L)²,  c₃ = c₂/2
L — динамический диапазон (255 для 8 bpp)
```

**Вычисление:** обычно в скользящем окне 11×11, итоговое SSIM = среднее по всем окнам (MS-SSIM).

**Диапазон:** SSIM ∈ [−1, 1], при идентичных изображениях SSIM = 1.

#### Недостатки SSIM

- Не всегда корректен для цветных изображений
- Может давать контринтуитивные результаты (изображение с неправильными цветами может иметь SSIM выше, чем зашумлённое, хотя визуально воспринимается хуже)
- Разработка показателей качества остаётся актуальной задачей
