## Шум

Шум (помеха) — случайные нежелательные изменения сигнала, возникающие во время захвата, хранения, передачи и обработки.

Обычно речь о высокочастотных шумах. Низкочастотный шум (дрейф среднего значения) в изображениях называют **неравномерностью фона**.

---

## Классификация шумов

### По типу наложения

| Тип | Модель | Особенность |
|-----|--------|-------------|
| Аддитивный (additive) | `S'(t) = S(t) + N(p₁,...,pₙ)` | Шумовая составляющая **не зависит** от значения сигнала |
| Мультипликативный (speckle) | `S'(t) = S(t) + S(t)·N(p₁,...,pₙ)` | Шумовая составляющая **зависит** от значения сигнала |
| Импульсный (salt & pepper) | `S'(t) = (1-p)S(t) + pN(...)`, p∈{0,1} | Только **часть** отсчётов изменяется, принимают мин/макс значения |

### По корреляции

- **Некоррелированный** (декоррелированный): шум пикселя не зависит от соседей
- **Коррелированный**: значение шума зависит от шума соседних пикселей

### По спектру (некоррелированный шум)

- **Белый** — спектральные составляющие распределены равномерно
- **Цветной** — неравномерное распределение (розовый, синий и др.)

---

## Аддитивный белый Гауссов шум (AWGN)

```
N(z) = 1/(√(2πσ)) · e^(-(z-z̄)²/(2σ²))
```

В AWGN: `z̄ = 0`

### Алгоритм Бокса-Мюллера (генерация нормального распределения N(z̄, σ²))

1. Генерировать независимые x и y, равномерно распределённые на [-1, 1]
2. Вычислить `s = x² + y²`. Если `s > 1` или `s = 0` → повторить шаг 1
3. Вычислить:
   - `z₀ = x · √(-2 ln s / s)`
   - `z₁ = y · √(-2 ln s / s)`
4. z₀ и z₁ — независимые случайные с. стандартного нормального распределения N(0,1). Для N(z̄, σ²): умножить на σ и прибавить z̄.

---

## Мультипликативный белый Гауссов шум (MWGN)

В MWGN: `z̄ = 1`, `σ² = σk² / (σk² + z̄)`

---

## Импульсный шум (salt and pepper)

```
P(z) = { Pₐ,  z = a
        { Pb,  z = b
        { 0,   иначе
```

Плотность вероятностей: `p(z) = Pₐ·δ(z-a) + Pb·δ(z-b)`

Каждый пиксель с вероятностью Ps заменяется на значение a или b (min/max).

---

## Другие модели шума

| Модель | p(z) | μ | σ² |
|--------|------|---|----|
| Равномерный | 1/(b-a), a≤z≤b | (a+b)/2 | (b-a)²/12 |
| Релея | (2/b)(z-a)e^(-(z-a)²/b), z≥a | a+√(πb/4) | b(4-π)/4 |
| Эрланга (гамма) | (a^b · z^(b-1) · e^(-az))/(b-1)!, z≥0 | b/a | b/a² |
| Экспоненциальный | ae^(-az), z≥0 | 1/a | 1/a² |

Шумы цифровых камер — пространственно коррелированные, не белые (автокорреляционная функция ≠ δ-функции).

## Артефакты компрессии

Сжатие с потерями (JPEG и др.) порождает:
- **Блочные артефакты** — заметные границы блоков 8×8
- **Эффект Гиббса (звон, ringing)** — осцилляции возле резких границ

Схема JPEG: YCC → блоки 8×8 → ДКП → квантование → зигзаг → энтропийное сжатие.

---

## Муар (moire pattern)

Паразитная текстура или регулярная помеха из-за наложения спектров. Возникает при сканировании растрированных при печати изображений.

---

## Нечёткость/размытость (blurring)

Причины: расфокусировка оптики, дрожание камеры. Проявляется в сглаживании профиля яркости на границах.

---

## Низкий контраст

**Контраст** = отношение яркостей самой светлой и самой тёмной частей. При низком контрасте гистограмма «узкая», динамический диапазон не используется.

---

## Искажения цветов (color cast)

Ошибочный баланс белого → сдвиг каналов R, G, B. Пример: yellow cast = мало синего.

---

## Неравномерный фон

Низкочастотный тренд, мешает выделению объектов при сегментации.

---

## Классификация фильтров

### Линейность

Фильтр **линейный**, если выполняется принцип суперпозиции:
```
f(aI₁ + bI₂) = af(I₁) + bf(I₂)
```

### По импульсной характеристике (линейные фильтры)

| Тип | Название | Формула |
|-----|----------|---------|
| КИХ (FIR) | Нерекурсивный | `y(n) = Σᵢ bᵢ·x(n-i)` |
| БИХ (IIR) | Рекурсивный | `y(n) = Σ bᵢ·x(n-i) - Σ aₖ·y(n-k)` |

### По АЧХ

| Тип | Английский | Описание |
|-----|-----------|----------|
| ФНЧ (LPF) | Low-pass filter | Пропускает низкие частоты |
| ФВЧ (HPF) | High-pass filter | Пропускает высокие частоты |
| Полосовой (BPF) | Band-pass filter | Пропускает полосу частот |
| Режекторный (BSF) | Band-stop filter | Подавляет полосу частот |

ФНЧ → сглаживание/размытие; ФВЧ → выделение границ.

Полосовые и режекторные = последовательное соединение ФНЧ + ФВЧ.

**ФНЧ**: сумма элементов ядра = 1, все элементы ≥ 0.  
**ФВЧ**: сумма элементов ядра = 0, присутствуют отрицательные элементы.

## Свёртка (1D, временная область)

```
s(n) = Σₘ₌₀^{M-1} a(m)·b(n-m)
```

Пример: a = [2,1,3,-1], b = [-1,1,2] → s = [-2,1,2,6,5,-2]

Свёртка — **линейный оператор** (суперпозиция выполняется).

Свойства:
- Коммутативность: `I * H = H * I`
- Ассоциативность: `I * (H * M) = (I * H) * M`
- Дистрибутивность: `I * (H + M) = I*H + I*M`

Свёртка реализует КИХ-фильтр; элементы b(i) — **коэффициенты фильтра** (ядро свёртки, convolution kernel).

Помимо линейной существует **циклическая (круговая) свёртка** — число отсчётов в сигнале и ядре одинаковое, ядро циклически переставляется.

---

## Свёртка 2D (пространственная область)

```
I'(r,c) = Σᵢ₌₁^K Σⱼ₌₁^L I(r+i-(K+1)/2, c+j-(L+1)/2) × H(i,j)
```

H(i,j) — **ядро свёртки** (маска), размер K×L. Ядро «помещается сверху» на изображение, элементы перемножаются и суммируются, результат → центральный элемент ядра.

### Пример C++ кода (свёртка для 8 bpp, граничные строки копируются)

```cpp
void Convolution2D(const unsigned char* pSrc, unsigned char* pRes, int iWidth,
                   int iHeight, const int* pKernel, int iKernelSize) {
    int iHalf = iKernelSize / 2;
    memcpy(pRes, pSrc, sizeof(*pRes) * iWidth * iHeight);
    for (int y = iHalf; y < iHeight - iHalf; ++y) {
        for (int x = iHalf; x < iWidth - iHalf; ++x) {
            const int* pk = pKernel;
            const unsigned char* ps = &pSrc[(y - iHalf) * iWidth + x - iHalf];
            int iSum = 0;
            for (int v = 0; v < iKernelSize; ++v) {
                for (int u = 0; u < iKernelSize; ++u)
                    iSum += ps[u] * pk[u];
                pk += iKernelSize;
                ps += iWidth;
            }
            if (iSum > 255) iSum = 255;
            else if (iSum < 0) iSum = 0;
            pRes[y * iWidth + x] = (unsigned char)iSum;
        }
    }
}
```

## Обработка граничных строк и столбцов

Для K/2 строк/столбцов с каждой стороны применяют один из подходов:

1. **Не фильтровать** — результат меньше исходного на K-1 строк и столбцов
2. **Копировать** граничные значения без изменений (тот же размер)
3. **Дополнить** изображение перед фильтрацией:
   - константой (нулём)
   - повторением крайних строк/столбцов
   - симметричным отражением
   - значениями, не изменяющими производную на границе

## Свёртка в частотной области

```
I * H = IFFT(FFT(I') × FFT(H'))
```

- I' — изображение, дополненное нулями до размера, кратного степени двух
- H' — ядро, дополненное нулями до размера I'

Алгоритм: FFT(I') → перемножить поэлементно с FFT(H') → IFFT.

---

## Сепарабельные фильтры

Двумерный фильтр **сепарабельный**, если 2D-фильтрацию можно заменить двумя 1D-фильтрациями (сначала по строкам, затем по столбцам).

Пример — **гауссиан**:

```
I₂ = G * I₁ = Σₕ e^(-h²/(2σ²)) · Σₖ e^(-k²/(2σ²)) · I₁(i-h, j-k)
```

---

## Эффективность способов свёртки

| Метод | Сложность |
|-------|-----------|
| Пространственная (2D) | O(M·N·K·L) |
| Сепарабельная | O(M·N·(K+L)) |
| Частотная (FFT) | O(N'² · log₂ N') |

При **больших ядрах** предпочтительнее свёртка в частотной области.
