## Спектр сигнала

**Спектр сигнала (спектральный образ)** — коэффициенты разложения сигнала в базисе ортогональных функций. Спектральное разложение обратимо, результат — комплексные числа.

Базисные функции: Фурье, Хаара, Уолша, Бесселя, Лежандра и др.

### Дискретное преобразование Фурье (ДПФ)

**Прямое ДПФ:**

$$X(k) = \frac{1}{N} \sum_{n=0}^{N-1} x(n) \exp\!\left(-j2\pi kn/N\right), \quad k = 0,1,\ldots,N-1$$

**Обратное ДПФ (ОДПФ):**

$$x(n) = \sum_{k=0}^{N-1} X(k) \exp\!\left(j2\pi kn/N\right), \quad n = 0,1,\ldots,N-1$$

---

## Три вида спектра

| Вид спектра | Формула |
|---|---|
| **Амплитудный** | $An(k) = \sqrt{\operatorname{Re}(X(k))^2 + \operatorname{Im}(X(k))^2}$ |
| **Фазовый** | $\Phi(k) = \operatorname{arctg}\!\left(\dfrac{\operatorname{Im}(X(k))}{\operatorname{Re}(X(k))}\right)$ |
| **Энергетический** | $En(k) = \operatorname{Re}(X(k))^2 + \operatorname{Im}(X(k))^2$ |

---

## Анализ амплитудного спектра

- Амплитудный спектр **симметричен** относительно 0-й частоты → часто показывают только половину
- Значение в 0-й частоте пропорционально **постоянной составляющей** (среднему значению сигнала)
- На спектре видны **преобладающие гармонические составляющие** (пики)

> Пример: сигнал из двух синусоид + белый шум даёт два пика на амплитудном спектре

---

## Примеры спектров одномерных сигналов

| Сигнал во времени | Спектр |
|---|---|
| Синусоида $X(t) = X_m \sin(\omega_0 t + \varphi)$ | Один пик на частоте $\omega_0$ |
| Прямоугольные импульсы (период T, длит. τ) | Гармоники на $\omega_0, 2\omega_0, 4\omega_0, 6\omega_0, \ldots$ (спад по sinc) |
| Треугольные импульсы | Аналогично, но спад быстрее |
| Одиночный прямоугольник | Непрерывный спектр вида sinc |
| sinc-функция $X(t) = X_m \dfrac{\sin(\pi t/T)}{\pi t/T}$ | Прямоугольный спектр шириной $2\pi/T$ |

> **Правило двойственности:** sinc во времени → прямоугольник в частоте; прямоугольник во времени → sinc в частоте

## БПФ — построение спектра

**Вычислительная сложность ДПФ:** $O(N^2)$

**Быстрое Преобразование Фурье (БПФ, FFT):** $O(N \log N)$

Алгоритмы: с **прореживанием по времени** (Decimation in Time, **DiT**) и **по частоте** (Decimation in Frequency, **DiF**) — алгоритмы Кули–Тьюки (Cooley–Tukey).

> Требование: размер массива должен быть **степенью двойки**. Если нет — дополнить нулями (zero-padding).

---

## БПФ с прореживанием по времени (DiT)

Идея: разбить сигнал на чётные и нечётные отсчёты:

$$X(k) = \sum_{n=0}^{N-1} x(n) W_N^{kn}, \quad W_N = \exp(-j2\pi/N)$$

$$X(k) = \underbrace{\sum_{n=0}^{N/2-1} x(2n) W_{N/2}^{nk}}_{X_1(k)} + W_N^k \underbrace{\sum_{n=0}^{N/2-1} x(2n+1) W_{N/2}^{nk}}_{X_2(k)}$$

$$X(k) = \begin{cases} X_1(k) + W_N^k X_2(k), & 0 \le k < N/2 \\ X_1(k - N/2) - W_N^{k-N/2} X_2(k - N/2), & N/2 \le k < N \end{cases}$$

### Операция «бабочка»

```
a ────────────────── A = a + b·W_N^r
         ╲  ╱
          ╳  × W_N^r
         ╱  ╲
b ────────────────── B = a - b·W_N^r
```

$W_N^r$ — поворачивающий множитель (twiddle factor).

### Схема 8-точечного БПФ (DiT)

- **Стадия 1:** 2-точечные БПФ (исходные данные в бит-реверсивном порядке)
- **Стадия 2:** объединение 2-точечных → 4-точечные
- **Стадия 3:** объединение 4-точечных → 8-точечный БПФ

Итого $\log_2 N$ стадий.

---

## Бит-реверсивный порядок

Для N=8 (3 бита):

| Нормальный индекс | Бинарно | Бит-реверс | Новый индекс |
|:---:|:---:|:---:|:---:|
| 0 | 000 | 000 | 0 |
| 1 | 001 | 100 | 4 |
| 2 | 010 | 010 | 2 |
| 3 | 011 | 110 | 6 |
| 4 | 100 | 001 | 1 |
| 5 | 101 | 101 | 5 |
| 6 | 110 | 011 | 3 |
| 7 | 111 | 111 | 7 |

```cpp
void ComplexBitReverse(complex<double>* data, int size)
{
    int middle = size/2, revSize = size - 1, j = 0;
    for (int i = 0; i < revSize; ++i)
    {
        if (i < j)
            swap(data[i], data[j]);
        int k = middle;
        while (k <= j) { j -= k; k /= 2; }
        j += k;
    }
}
```

---

## Пример кода БПФ (C++)

```cpp
void FftDit(complex<double>* data, int size, int sizeLog2, int dir)
{
    ComplexBitReverse(data, size);
    int ptsInLeftDft, ptsInRightDft = 1;
    for (int stage = 1; stage <= sizeLog2; ++stage)
    {
        ptsInLeftDft = ptsInRightDft;
        ptsInRightDft *= 2;
        complex<double> twiddle(1.0, 0.0);
        double trigArg = M_PI / ptsInLeftDft;
        // dir == 1 для прямого, dir == -1 для обратного
        complex<double> wFactor(cos(trigArg), -sin(trigArg)*dir);
        for (int butterflyPos = 0; butterflyPos < ptsInLeftDft; ++butterflyPos)
        {
            for (int topNode = butterflyPos; topNode < size; topNode += ptsInRightDft)
            {
                int botNode = topNode + ptsInLeftDft;
                complex<double> temp = data[botNode] * twiddle;
                data[botNode] = data[topNode] - temp;
                data[topNode] += temp;
            }
            twiddle *= wFactor;
        }
    }
}
```

## fftshift — сдвиг постоянной составляющей

При вычислении ДПФ половина значений спектра — зеркальная копия (избыточна).

**Проблема:** постоянная составляющая (нулевая частота) находится в начале/конце массива.

**Решение — fftshift:** переставить вторую половину спектра в начало (и наоборот).

- Для **сигнала:** умножить нечётные отсчёты на $(-1)$: $x(n) \times (-1)^n$
- Для **изображения:** умножить пиксели на $(-1)^{r+c}$ перед ДПФ, или переставить квадранты после

> После fftshift постоянная составляющая (DC) оказывается в центре спектра.

## Спектрограмма (STFT)

**Short-Time Fourier Transform (STFT):** сигнал разбивается на (перекрывающиеся или нет) окна, для каждого окна вычисляется ДПФ.

**Спектрограмма (сонограмма)** — визуализация: ось X = время, ось Y = частота, яркость/цвет = амплитуда.

Содержит 3D-данные, отображается как изображение.

---

## Спектр изображения (2D DFT)

### Прямое двумерное ДПФ

$$F(u,v) = \frac{1}{\sqrt{N \times M}} \sum_{r=0}^{N-1}\sum_{c=0}^{M-1} f(r,c)\exp\!\left(-j2\pi ur/N\right)\exp\!\left(-j2\pi vc/M\right)$$

### Обратное двумерное ДПФ

$$f(r,c) = \frac{1}{\sqrt{N \times M}} \sum_{u=0}^{N-1}\sum_{v=0}^{M-1} F(u,v)\exp\!\left(j2\pi ur/N\right)\exp\!\left(j2\pi vc/M\right)$$

где $r,u = 0,\ldots,N-1$; $c,v = 0,\ldots,M-1$.

В спектре изображения:
- Центр (после fftshift) — **НЧ-компоненты** (низкие частоты, общий фон)
- Края — **ВЧ-компоненты** (высокие частоты, детали, границы)

---

## Построение спектра изображения

Сложность наивного 2D ДПФ: $O(N^4)$ для изображения $N \times N$.

**Сепарабельность ДПФ:** можно вычислить одномерные БПФ по строкам, затем по столбцам.

**Алгоритм для полутонового изображения:**
1. Дополнить строками и столбцами нулей до размера $Z \times Z$, где $Z$ — степень 2, $Z \ge N$
2. Выполнить БПФ для каждой строки
3. Выполнить БПФ для каждого столбца
4. Переставить квадранты (fftshift) для сдвига DC в центр

**Итоговая сложность:** $O((Z \log Z)^2)$

---

## fftshift для изображений

**До fftshift:** DC (нулевая частота) — в левом верхнем углу.

**После fftshift:** DC — в центре изображения (переставляются 1-й квадрант с 3-м, 2-й с 4-м).

Реализация через умножение: $f(r,c) \times (-1)^{r+c}$ до ДПФ.

---

## Примеры спектров изображений

| Изображение | Спектр |
|---|---|
| Синусоидальная сетка (2 горизонт. + 1 вертик.) | Отчётливые пики на соответствующих частотах |
| Текстура (кирпичная кладка) | Пики вдоль доминирующих направлений |
| Гауссиан с малой дисперсией | Гауссиан с **большой** дисперсией в спектре |
| Гауссиан с большой дисперсией | Гауссиан с **малой** дисперсией в спектре |
| Вертикальная полоска | Горизонтальная яркая линия в спектре |
| Диагональная полоска | Линия под углом 90° к оригиналу |

> Ориентация наиболее яркой линии в спектре **перпендикулярна** ориентации объектов на изображении (90°).

## Кепстр сигнала

**Кепстр (cepstrum)** — результат ОДПФ логарифма энергетического спектра:

$$C_p = \left|\mathcal{F}^{-1}\!\left\{\log\!\left(|\mathcal{F}\{f(t)\}|^2\right)\right\}\right|^2$$

По смыслу — «спектр спектра».

**Применение:** выявление периодических составляющих в спектре (например, эхо).

- Ось аргумента кепстра называется **quefrency** (по аналогии с frequency)
- Название «кепстр» — перестановка букв слова «спектр»

**Пример:** у голосового сигнала кепстр выявляет период основного тона (pitch period).

---

## Кепстр изображения

Аналогично вычисляется кепстр двумерного изображения.

**Применение:** оценка **PSF (Point Spread Function, функции рассеяния точки)** при размытии движением (motion blur).

По кепстру размытого изображения или кепстру его градиента можно определить:
- направление смаза
- величину смаза

Это позволяет восстановить оригинальное изображение (деконволюция).
