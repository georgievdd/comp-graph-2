# Лекция 9. Сегментация изображений

## Что такое сегментация

**Сегментация** — разбиение изображения на области (сегменты) с одинаковыми свойствами.

- **Семантическая сегментация**: каждый пиксель относится к классу (все пиксели класса — один сегмент)
- **Instance segmentation**: каждый отдельный объект класса — свой сегмент

Подходы:
- **Индуктивный** (снизу вверх): объединяем похожие пиксели
- **Дедуктивный** (сверху вниз): задаём модель сегмента, ищем соответствия

---

## Пороговая сегментация (бинаризация)

Если гистограмма **бимодальная** (два пика), порог ставят в «долине» между пиками:

```
Binary(x,y) = { 1, I(x,y) ≥ T
               { 0, I(x,y) < T
```

---

## Алгоритм Оцу

Задача: найти порог T, минимизирующий **внутриклассовую** дисперсию или, эквивалентно, максимизирующий **межклассовую**:

```
σ²w(t) = ω₀·σ²₀ + ω₁·σ²₁    → минимум
σ²b(t) = ω₀·ω₁·(μ₀ - μ₁)²   → максимум
```

- ω₀, ω₁ — вероятности классов (относительные частоты)
- μ₀, μ₁ — средние яркости классов
- σ²₀, σ²₁ — дисперсии классов

---

## Соседство пикселей

**4-связность**: соседи по 4 направлениям (вверх, вниз, влево, вправо)

**8-связность**: соседи по 8 направлениям (включая диагонали)

При маркировке объектов: если объект 4-связный, то фон 8-связный и наоборот.

---

## Маркировка связных компонент (CCL)

### Алгоритм заливки с затравкой (FloodFill)

Для каждого непомеченного пикселя объекта:
1. Назначить новую метку
2. Добавить в стек
3. Пока стек не пуст: извлечь пиксель, пометить соседей той же меткой, добавить в стек

### Построчный алгоритм заливки

1. Найти затравку (seed) → добавить в стек интервал строки [X_left, X_right, y]
2. Пока стек не пуст:
   - Извлечь интервал (xl, xr, y)
   - Залить строку y от xl до xr
   - Найти все интервалы в строках y-1 и y+1, частично перекрывающих [xl,xr], добавить в стек

---

## Двухпроходный алгоритм разметки (Two-Pass CCL)

Бинарное изображение сканируется слева направо и сверху вниз:

1. Если текущий пиксель — объект: проверить уже просмотренные соседи (зависит от 4- или 8-связности):
   - Нет размеченных соседей → новая метка
   - Один размеченный сосед → присвоить его метку
   - Несколько разных меток → присвоить наименьшую, записать остальные в таблицу эквивалентности

2. Второй проход: заменить эквивалентные метки на одинаковые

**На практике** эквивалентные метки часто переназначают уже в первом проходе, а таблица эквивалентности не создаётся.

---

## Алгоритм разбиения/слияния (Split/Merge)

Два критерия **однородности** (homogeneity) области: для разбиения (split) и слияния (merge).

Обычно критерии — пороговые ограничения для дисперсии или разности max−min.

**Алгоритм** (квадродерево):
1. Если прямоугольная область **неоднородна** → разделить на 4 равные подобласти. Повторять пока есть неоднородные.
2. Попарно соседние области: если объединённая будет **однородной** → объединить. Повторять пока есть такие пары.

**Недостатки**:
- Результат зависит от порядка выбора областей для слияния
- Сложно подобрать критерии: oversegmentation (много малых) или undersegmentation (разные объекты слиты)

---

## Разбиение на суперпиксели (Superpixels)

Развитие идеи split/merge. Суперпикселей — области произвольной формы, для которых выполняются критерии однородности и компактности.

Для сегментации суперпиксели объединяют по критерию схожести (similarity) с помощью **иерархической агломерации по графу смежности** (Region Adjacency Graph, RAG).

---

## Алгоритм водораздела (Watershed)

Изображение рассматривается как рельеф местности. В местах **локальных минимумов** образуются водосборные бассейны. Бассейны заполняются. В пикселях, где встречаются воды из соседних бассейнов, строится **граница**.

**Маркерный водораздел** (marker-controlled watershed): чтобы избежать чрезмерной сегментации, выбирают инициирующие точки (маркеры) бассейнов.

**Применение для слипшихся частиц**:
1. Бинаризация
2. **Distance transform**: каждый пиксель = евклидово расстояние до ближайшего фонового пикселя
3. Водораздел на инвертированном distance transform → линии разделяют частицы

---

## Сегментация как задача машинного обучения

**Обучение без учителя** (unsupervised): k-means, mean-shift — кластеризация в пространстве признаков.

**Обучение с учителем** (supervised): kNN, SVM, Random Forest, нейронные сети. Требует размеченной выборки (обучающая / валидационная / тестовая).

При **глубоком обучении**: в выборке хранятся изображения + разметка (ground truth).

---

## Метод k-средних (k-means)

Минимизирует суммарное квадратичное отклонение точек от центров кластеров.

**Алгоритм**:
1. Задать k, случайно выбрать k центроид
2. Отнести каждую точку к ближайшей центроиде (евклидово расстояние)
3. Пересчитать центроиды (координаты центра масс кластера)
4. Повторять 2-3 до сходимости (нет изменений в составе кластеров)

**Метод локтя (elbow)**: перебрать диапазон k, вычислить WCSS (Within Cluster Sum of Squares). Оптимальное k — в месте резкого излома графика WCSS(k).

**Недостатки k-means**:
- Не гарантируется глобальный минимум (только локальный)
- Результат зависит от начальных центроид
- Число кластеров нужно знать заранее

**Модификации**: k-means++, k-medians, fuzzy c-means.

## Сегментация с помощью классификации (пример)

Задача: сегментация текста, иллюстраций и фона на страницах журналов.

1. Создать выборку из сотен изображений, разметить вручную на 3 класса
2. Разбить изображение на перекрывающиеся блоки N×N, для каждого вычислить признаки:

```
Среднее интенсивности:  B̄ᵢ = (ΣΣ Bᵢ(r,c)) / N²

СКО: σᵢ = √(ΣΣ(B̄ᵢ - Bᵢ(r,c))² / N)

Среднее разностей средних с 4 соседями: dBᵢ = Σ|B̄ᵢ - B̄ₖ| / 4

Средний модуль градиента: Gⁱ = ΣΣ ∇Bᵢ(r,c) / (N-2)²
```

3. Обучить 3 классификатора (one-to-rest, Random Forest)
4. Подобрать параметры на валидационной выборке
5. Сегментация: вычислить признаки → классификаторы → сгладить вероятности по соседним блокам → голосование
